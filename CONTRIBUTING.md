First off, thank you for considering contributing!

## How Can I Contribute?

### Reporting Bugs

If you find a bug, please make sure it hasn't already been reported by searching the [Issues](https://github.com/PytatoDuck/ratio-lab/issues) on GitHub.

If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/PytatoDuck/ratio-lab/issues/new). Be sure to include a **title and clear description**, as much relevant information as possible, and a **code sample or an executable test case** demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements

If you have an idea for an enhancement or a new feature, please open an issue to discuss it first.

### Your First Code Contribution

Unsure where to begin contributing? You can start by looking through `good first issue` and `help wanted` issues:
*   **Good first issues** - issues which should only require a few lines of code, and a test or two.
*   **Help wanted** - issues which should be a bit more involved than `good first issue` issues.

## Pull Request Process

We follow the "fork-and-pull" Git workflow.

1.  **Fork** the repo on GitHub.
2.  **Clone** your **fork** locally
3.  **Create a new branch** for your changes: `git checkout -b name-of-your-feature-or-fix`
4.  Make your changes and **commit** them with a clear, descriptive message.
5.  **Push** your branch to your fork on GitHub: `git push origin name-of-your-feature-or-fix`
6.  Open a **Pull Request** from your fork to **this** repository.

Thank you again for your contribution!
